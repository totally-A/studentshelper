Оператор вибору if else

У деяких джерелах говориться, що оператор вибору if else - самостійний оператор. Але це не так, if else - це всього лише форма запису оператора вибору if. Оператор if else дозволяє визначити програмісту дію, коли умова істинна і альтернативну дію, коли умова помилкова. Тоді як if дозволяв визначити дію при істинній умові. Синтаксис запису оператора вибору if else: 

if (/ * перевіряється умова * /)
	{
		/ * Тіло оператора вибору 1 * /;
	} 
	else
		{
			/ * Тіло оператора вибору 2 * /;
		}

Читається так: «Якщо перевіряється умова істинно, то виконується тіло оператора вибору 1, інакше (тобто перевіряється умова помилкова) виконується тіло оператора вибору 2«. Зверніть увагу на те, як записано оператор if else. Слово else спеціально зрушено вправо для того щоб програмний код був зрозумілий і його було зручно читати.

Гарний стиль програмування

  • Одноразове використання узгоджень про відступи значно підвищує читабельність програми. Ми пропонуємо використовувати для відступів табуляцію фіксованого розміру приблизно в ¼ дюйма або три пробіли.
  • Робіть відступи для обох груп операторів структури if/else.
  • При наявності декількох рівнів відступів, відступи для кожного рівня повинні містити одну і ту саму кількість додаткових пробілів.
  • Деякі програмісти віддають перевагу вводити початкову і кінцеву фігурні дужки у складових операторах до початку введення оператора в середині фігурних дужок. Це допомагає уникнути пропусків одної або обох фігурних дужок.

Приклад програми

Розглянемо невеликий код програми. Перед користувачем з’являється вікно, в якому йому пропонується зайти до певної гри, завантажити гру, зайти до мультиплеєру, а також можна вийти з програми. Потрібно вибрати щось одне. Це типовий приклад оператору if else. Маємо код задачі: 

//defines the entry point for the console application
#include< stdio.h >
#include< iostream >

using namespace std;

int main()
{
	system("color F0");

	int input;

	printf("1. Play a game\n");
	printf("2. Load a game\n");
	printf("3. Go to a multiplayer\n");
	printf("4. Exit\n");
	printf("Вибiр:\n");

	scanf("%d", &input);
	if (input == 1) printf("Starting a game...\n");
	else if (input == 2) printf("Game is loading...\n");
	     else if (input == 3) printf("Going to a multiplayer...\n");
		      else if (input == 4) printf("Thank you for the game!");
			       else printf("Try again!");
	system("pause");
}

Оператор множинного вибору switch

Отже ми розглянули оператор з одиночним вибором if і оператор з подвійним вибором if else, але у мові С ще є оператор множинного вибору switch, який ми зараз детально розглянемо. 

switch (/ * змінна або вираз * /)
 	{
 	case / * константне вираження1 / *:
 		{
 			/ * Група операторів * /;
 			break;
 		}
 	case / * константне вираженіе2 * /:
 		{
 			/ * Група операторів * /;
 			break;
 		}
 			//. . .
 	default:
        {
            / * Група операторів * /;
        }
 	}

На початковому етапі аналізується вираз або змінна. Після чого здійснюється перехід до тієї гілки програми, для якої значення змінної або виразу співпадає з константним виразом. Далі виконується оператор або група операторів поки не зустрінеться зарезервоване слово break або закриває фігурна дужках. Якщо значення змінної або виразу не збігається ні з одним константним виразом, то передається керування гілці програми, що містить зарезервоване слово default. Після чого виконується оператор або група операторів даної гілки. Зараз розглянемо задачу з використанням оператора вибору switch.

Гарний стиль програмування

  • Не забувайте про блок default в операторах switch. Випадки, які явно не перевіряються в структурі switch – ігноруються. Блок default допомагає уникнути цього, фокусуючи увагу програміста на необхідності обробляти не типові умови. Існують ситуації, коли ніякої обробки по замовчуванню (у блоці default) це потрібно.
  • Хоча case і default у структурі switch можуть виникати можуть виникати у довільному порядку, хорошим стилем програмування вважається розміщення default в кінці структури.
  • В тому випадку, коли в структурі switch default указано останнім, оператора break для нього не потрібен. Однак деякі програмісти вводять цей break для ясності та одноманітності з мітками case.
  • Не забувайте про необхідність обробки символів нового рядка у вхідному потоці при зчитуванні символів по одному.

Приклад програми

Умова даної задачі така сама, як і бeло розглянуто задачу у розділі if else, тобто Перед користувачем з’являється вікно, в якому йому пропонується зайти до певної гри, завантажити гру, зайти до мультиплеєру, а також можна вийти з програми. Потрібно вибрати щось одне. Маємо код: 

//defines the entry point for the console application
#include< stdio.h >
#include< iostream >

using namespace std;

int main()
{
	system("color F0");

	int input;

	printf("1. Play a game\n");
	printf("2. Load a game\n");
	printf("3. Go to a multiplayer\n");
	printf("4. Exit\n");
	printf("Choice:\n");

	scanf("%d", &input);
	switch (input) {
	case 1:            
		printf("Starting a game...\n");
		break;
	case 2:            
		printf("Game is loading...\n");
		break;
	case 3:            
		printf("Going to a multiplayer...\n");
		break;
	case 4:            
		printf("Thank you for the game!");
		break;
	default:            
		printf("Try again!");
		break;
	}
	system("pause");
}

Проте, звичайно, було б цікаво порівняти оператор if else та оператор switch, адже вони дійсно схожі між собою.

  • В else if контроль проходить через кожне if, до тих пір, поки не знайде істинне значення або йде до кінця. У випадку switch, відповідно до значення, управління переходить до відповідного випадку.
  • Switch є більш компактним, ніж багато вкладених один в одного if else. Таким чином, switch вважається більш зручним для читання.
  • Використання оператору break у switch має важливе значення, але немає необхідності у використанні break у if else.
  • У операторі switch для перемикання зазвичай використовується цілочисловий тип даних, тоді як у if else цей тип даних виключається, як характерний.
  • Ще одна відмінність між перемикачем і if else є те, що оператор перемикач вважається менш гнучким, ніж if else, так як він дозволяє тільки тестування одного виразу за списком дискретних значень.
  • Так як компілятор може оптимізувати оператор switch, він, як правило, вважається більш ефективним. Кожен case у switch не залежить від попереднього. У if else інакше, код повинен бути оброблений в порядку, визначеному програмістом.
  • Оператор switch працює на основі рівності, в той час як оператор if else працює на основі знаходження істинності чи хибності.
