Типові помилки програмування

  • Використання ком замість крапки з комою в заголовках структури for.
  • Розміщення крапки з комою відразу після заголовку структури for робить тіло цієї структури пустим оператором. Звичайно це є логічною помилкою.
// форма записи оператора цикла for:

for (/*выражение1*/; /*выражение2*/; /*выражение3*/)
{
	/*один оператор или блок операторов*/;
}

Якщо в тілі циклу for повинен виконається один оператор, тоді фігурні дужки можна опустити:

for (/*выражение1*/; /*выражение2*/; /*выражение3*/)
/*один оператор*/;

Розглянемо детально три вирази записані в круглих дужках циклу for.

Вираз 1 - оголошення (і) або ініціалізація, раніше оголошеної, змінної-лічильника, яка буде відповідати за істинність умови в циклі for. Приклад:

int counter = 0; //звідси видно, що була оголошена змінна counter типу int і ініціалізована значенням 0

Змінна-лічильник завжди повинна мати цілочисельний тип даних. Якщо змінна була оголошена в циклі (все одно в якому), по завершенні циклу ця змінна буде знищена.
Різниця між оголошенням та ініціалізацією змінної: 

counter; //оголошення змінної count

counter = 9; // ініціалізації цілочисельної змінної count значенням 9.

Вираз 2 - це умова продовження циклу for, воно перевіряється на істинність.

counter < 10; // умова істинна поки count строго менше десяти!

Вираз 3 змінює значення змінної-лічильника. Без виразу 3 цикл вважається нескінченним, так як зміна вмісту змінної count виконуватися не буде, і якщо спочатку умова була істинним, то цикл буде нескінченним, інакше програма навіть не увійде в цикл. Вирази 1, 2, 3 відокремлюються одна від одної обов'язковими роздільниками, крапкою з комою. Тіло циклу обрамляється фігурними дужками, якщо тіло циклу складається з одного оператора, то фігурні скобочки не потрібні. Під зміною значення змінної мається на увазі зменшення або збільшення значення, наприклад:

for (int counter = 0; counter < 15; counter++) //виконується приріст змінної counter с кроком 1 від 0 до 15

++ Це операція інкремента, збільшення значення змінної на одиницю;
-- Це операція декремента, зменшення значення змінної на одиницю.

Дуже часто неправильно інтерпретується запис меж в циклі for, в нашому випадку приріст змінної counter виконується з кроком 1 від 0 до 15. Зверніть увагу на кінцеву межа. У разі продовження циклу стоїть знак відносини строго менше, а значить, коли значення в змінної counter дорівнюватиме 14, виконається вихід з циклу.

Приклад програми 

#include < iostream >
using namespace std;

int main()
{
	system("color F0");
	for (int counter = 0; counter < 15; counter++)      //start of the cycle
		cout << " " << counter;                         //body of the cycle
	cout << endl;
	system("pause");
}

Цикл for записаний без фігурних дужок. А раз немає фігурних дужок, значить, тілом циклу є наступний оператор, після заголовка циклу, в нашому випадку - це оператор виведення: 

cout << " " << counter;                         //body of the cycle

Фігурні дужки можна не опускати, це кому як зручно. Але в даному випадку без фігурних дужок запис коду більш компактна, ніж з дужками.
